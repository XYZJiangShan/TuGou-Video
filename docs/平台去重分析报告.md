> 调研时间：2026-04-04 | 适用于土狗视频工具箱去重方案设计

## 一、各平台视频压缩/转码参数

### 1. 抖音（Douyin）

| 参数 | 值 |
|------|------|
| 编码格式 | H.264 (AVC) 为主，部分场景 H.265 (HEVC) |
| 服务端转码 | 上传后强制重编码 |
| 分辨率 | 720P / 1080P（服务器存储多档，按网络自适应下发） |
| 码率 | 约 2000-4000 kbps（1080P），移动端优先流畅度 |
| 帧率 | 一般 30fps CFR |
| 音频 | AAC，128-192kbps |
| 特殊机制 | 上传即重编码，MD5必然改变；服务器存储多个分辨率版本 |

### 2. 快手（Kuaishou）

| 参数 | 值 |
|------|------|
| 编码格式 | H.264 (AVC) 为主，推荐 HEVC（权重加成） |
| 服务端转码 | 上传后重编码，Atlas转码引擎 |
| 分辨率 | 推荐 1080×1920（竖屏），支持 1440×2560 超采样 |
| 码率 | 约 1500-3500 kbps（自适应），高清 VBR 12-15Mbps |
| 帧率 | 30fps 为主，支持 60fps |
| 音频 | AAC，建议 192kbps |
| 特殊机制 | 25fps+720p 组合可能触发风控降权；超过 15Mbps 码率会被平台压缩 |

### 3. 小红书（Xiaohongshu）

| 参数 | 值 |
|------|------|
| 编码格式 | H.264 (AVC)，支持 H.265 |
| 服务端转码 | 上传后压缩，网页端压缩程度 < 移动端 |
| 分辨率 | 推荐 1080P（1080×1920），最高支持 4K |
| 码率 | 推荐 10-20 Mbps 上传（平台会压到 3000-5000 kbps） |
| 帧率 | 30fps 常规，60fps 动态内容 |
| 音频 | AAC |
| 特殊机制 | 强调"信息密度"和"完播率"；网页端上传画质优于App端；超 1080P 建议网页端 |

### 4. B站（Bilibili）

| 参数 | 值 |
|------|------|
| 编码格式 | H.264 (AVC) 为主，支持 HEVC、AV1 |
| 服务端转码 | **存在"二压"机制**：超过码率阈值会被重新编码 |
| 分辨率 | 支持 720P ~ 8K |
| **不二压阈值** | **1080P 30fps: ≤3000kbps**；**1080P 60fps: ≤6000kbps** |
| 帧率 | 30fps / 60fps |
| 音频 | AAC，16bit，≤320kbps（32bit 会触发二压） |
| 特殊机制 | 峰值码率 ≤8500kbps；Level 5；关键帧间隔 ≥10秒；色彩空间 YUV420 8bit |

## 二、各平台去重检测机制

### 1. 抖音 —— 三重检测体系（2025年升级版）

```
上传视频 → MD5校验 → 帧级指纹检测 → 音频波纹识别 → 语义理解 → 判定
```

**层级1：元数据检测**
- **MD5哈希校验**：最基础的一层，完全相同的文件秒判重复
- **文件属性**：时长、分辨率、码率等元数据比对

**层级2：视频帧指纹检测（核心）**
- **关键帧抽取**：在第2秒、3秒、5秒、8秒、12秒等特定时间点抽帧
- **感知哈希（pHash）**：对关键帧计算感知哈希值
- **余弦相似度 + 汉明距离**：与数据库中已有视频比对
- **2025年精度**：帧级指纹精度提升至 99.7%

**层级3：音频指纹检测**
- **音频波纹识别**：覆盖 99.2% 相似度检测
- **MFCC特征**：梅尔频率倒谱系数分析

**层级4：语义级内容理解（2025年新增）**
- **大模型判断**：首次引入大模型进行内容语义理解
- **文案/字幕识别**：OCR+NLP 分析画面文字
- **行为特征**：账号上传模式、设备指纹等

### 2. 快手 —— "第三代风铃系统"

```
上传视频 → 0.08秒全维度扫描 → 3D检测模型判定
```

**画面特征检测**
- **残差网络（ResNet）** 提取关键帧 DCT系数
- 比对色块分布与运动矢量
- 关键帧位移算法切割 150-200 个微片段

**音频指纹检测**
- **MFCC 梅尔倒谱系数**：每 0.25秒 分析声纹波动曲线
- 音频哈希值比对

**文本语义检测**
- **ERNIE 3.0 多模态模型**
- 识别字幕、封面文字、语音转译内容的语义重合度

**合规阈值**
- 同一场景素材复用间隔 > 72小时
- 单账号素材重复率 < 18%
- 二次创作需 > 7处实质性修改，原创元素占比 > 40%

### 3. 小红书 —— 三重校验机制

```
上传内容 → 特征值比对 → 语义相似度检测 → 账号关联图谱 → 判定
```

**特征值比对系统（0.05秒级）**
- **MD5 值校验**
- **HSV 色彩模型**：分析色相/饱和度/明度分布
- **LBP（局部二值模式）**：纹理特征提取
- 三重校验并行执行

**语义相似度检测**
- **BERT + TextRank 混合模型**
- 计算标题和正文的向量空间相似度
- **阈值 62%** 触发预警

**账号关联图谱**
- 32个维度建立账号关联性模型
- 设备指纹、网络环境、操作行为
- 重点监控批量账号的协同动作

### 4. B站 —— "撞车系统"（工业级方案）

```
上传视频 → 预处理(切黑边) → ResNet50特征提取 → Faiss向量检索 → 霍夫变换精排 → 判定
```

**预处理**
- **核心内容框提取器**：边缘检测 → 矩形裁剪 → 去除黑边/画中画
- GPU加速（NvCodec SDK + CUDA Kernel）

**图像特征提取**
- **ResNet50**（经 ViT 知识蒸馏优化）
- 自监督训练：随机数据增强模拟各种编辑手段
- 特征量化（PQ32）降低存储

**音频特征提取**
- **Log-FilterBank + MFCC**
- C++实现，比Python快10倍

**检索匹配**
- **粗筛**：Faiss 近似近邻检索（数十亿级索引库）
- **精排**：霍夫变换计算相似片段评分 + NMS
- **性能**：96%视频10秒内完成查重
- **准确率**：约 88%

## 三、针对各平台的土狗去重方案

### 🔥 方案设计原则

```
核心思路：让平台的特征提取结果与原视频产生差异
         但人眼观看时差异不可感知或可接受
```

### 方案A：抖音去重（难度：⭐⭐⭐⭐）

抖音检测最严格（帧级精度99.7%+语义理解），需要多维度同时修改：

| 处理项 | 技术手段 | FFmpeg 参数思路 |
|--------|----------|----------------|
| **帧级指纹破坏** | 关键帧时间点偏移 | 修改 GOP 结构、插入/删除首尾帧 |
| **画面微调** | 亮度/对比度/饱和度微调 | `eq=brightness=0.02:contrast=1.02:saturation=1.05` |
| **分辨率变换** | 上采样到1080P/微调分辨率 | `scale=1920:1080:flags=lanczos` 或 `scale=1078:1918` |
| **帧替换（v3方案）** | 33%帧做模糊+色偏+噪点替换 | boxblur+hue+noise 条件滤镜 |
| **镜像/轻微裁剪** | 画面水平翻转或边缘裁剪2-4px | `hflip` 或 `crop=iw-4:ih-4:2:2` |
| **音频指纹破坏** | 微调音高/速度/加底噪 | `asetrate=44150,aresample=44100` + `anoisesrc` 混合 |
| **重编码** | 不同编码参数重新编码 | 不同 CRF/码率/preset |

**推荐组合（抖音终极方案）：**
```
帧替换v3 + 1080P上采样 + 画面微调(亮度/饱和度) + 音高微调 + 重编码
```

### 方案B：快手去重（难度：⭐⭐⭐⭐）

快手使用 DCT系数 + MFCC + ERNIE3.0，检测速度极快（0.08秒）：

| 处理项 | 技术手段 | FFmpeg 参数思路 |
|--------|----------|----------------|
| **DCT系数干扰** | 画面噪点注入 + 模糊 | `noise=alls=3:allf=t` + `unsharp=3:3:0.3` |
| **色彩偏移** | HSV空间色温微调 | `hue=h=2` + `colorbalance=rs=0.02:gs=-0.01` |
| **运动矢量干扰** | 帧率微调/帧替换 | 帧替换v3方案（替换33%帧） |
| **音频声纹混淆** | 基频偏移±15Hz + 底噪 | `asetrate=44115` + `-26dB` 环境噪声混入 |
| **画面结构变化** | 轻微缩放+padding | `scale=iw*1.02:ih*1.02,crop=iw:ih` |
| **重编码** | 不同编码参数 | HEVC 编码（快手有权重加成） |

**推荐组合（快手方案）：**
```
帧替换v3 + 色彩偏移 + 音频基频偏移 + HEVC重编码 + 噪点注入
```

### 方案C：小红书去重（难度：⭐⭐⭐）

小红书侧重 HSV色彩模型 + LBP纹理 + 语义相似度：

| 处理项 | 技术手段 | FFmpeg 参数思路 |
|--------|----------|----------------|
| **HSV特征破坏** | 色相旋转 + 饱和度调整 | `hue=h=3:s=1.05` |
| **LBP纹理破坏** | 轻微锐化/模糊交替 | `unsharp=3:3:0.5` 或 `boxblur=1:1` |
| **分辨率适配** | 输出1080×1920竖屏 | `scale=1080:1920:flags=lanczos` |
| **MD5破坏** | 重编码即可 | 任何重编码都会改变MD5 |
| **画面结构** | 加边框/水印/片头片尾 | `pad=iw+20:ih+20:10:10:black` |
| **码率优化** | 15Mbps上传（网页端） | `-b:v 15M` |

**推荐组合（小红书方案）：**
```
色相偏移 + 轻微锐化 + 1080P输出 + 重编码 + 可选片头片尾
```

### 方案D：B站去重（难度：⭐⭐⭐⭐⭐）

B站的 ResNet50 + Faiss 是工业级方案，88%准确率，最难绕过：

| 处理项 | 技术手段 | FFmpeg 参数思路 |
|--------|----------|----------------|
| **ResNet特征干扰** | 帧替换v3（核心） | 模糊+色偏+噪点替换33%帧 |
| **黑边检测绕过** | 确保无黑边/适当裁剪 | `crop=in_w:in_h-2:0:1` |
| **画面变换** | 轻微旋转+缩放 | `rotate=0.01` + `scale=iw*1.01:ih*1.01` |
| **音频特征干扰** | MFCC特征修改 | 音高微调 + 混入环境音 |
| **编码适配** | 走不二压通道 | H.264, ≤6000kbps(60fps), Level 5, YUV420 8bit |
| **片段匹配干扰** | 调整视频速度/插入过渡 | `setpts=0.98*PTS` 或插入1秒渐变过渡 |

**推荐组合（B站方案）：**
```
帧替换v3 + 画面微调(亮度/色相/噪点) + 音频修改 + 不二压参数编码
```
